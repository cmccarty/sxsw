Trip Advisor API Examples

Below are examples for all of the API endpoints made available through the Trip Advisor API.

The core operations are:
- Location (/location)
- Reviews (/reviews)
- Nearby (/map)
- Search (/search)

Search allows you to search by keywords for matching locations (hotels, restaurants, attractions, or geos). You can restrict searches to a particular geo (eg Austin), as well as certain category types (eg. restaurants).

Nearby allows you to specify a lat/lon pair to search near. You can set a radius if you like, and order by proximity or popularity. You can also add category types, a geo, or a search query to further filter.

Location allows you to get more information about a location. If you don't specify any connections, you will get meta data (name, address, num_reviews, ratings, etc). You can also specify /attractions, /restaurants, /hotels to list other location types within a geo. You can get more results by paging with offset, limit.

Docs

You can find further documentation here: http://api.tripadvisor.com/api/mobex/1.1/doc?key=[API_KEY]
